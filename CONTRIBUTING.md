## 🔄 Как внести изменения

### 1. Форк репозитория
Создайте форк основного репозитория в свой аккаунт GitHub/GitLab.

### 2. Клонируйте форк
```bash
git clone https://github.com/ваш-username/sirius-rental.git
cd sirius-rental
```

### 3. Создайте ветку для изменений
```bash
git checkout -b feature/название-фичи
# или
git checkout -b fix/название-исправления
```

**Правила именования веток:**
- `feature/` — новые функции
- `fix/` — исправления ошибок
- `docs/` — изменения в документации
- `refactor/` — рефакторинг кода

### 4. Внесите изменения
Следуйте стандартам кода, описанным ниже.

### 5. Запустите тесты
```bash
pytest tests/test_api.py -v
```

Все тесты должны проходить перед отправкой изменений.

### 6. Закоммитьте изменения
```bash
git add .
git commit -m "feat: добавлена возможность поиска свободных комнат"
```

**Формат коммитов:**
- `feat:` — новая функция
- `fix:` — исправление ошибки
- `docs:` — изменения в документации
- `style:` — форматирование кода
- `refactor:` — рефакторинг
- `test:` — добавление тестов
- `chore:` — обновление зависимостей

### 7. Отправьте ветку в форк
```bash
git push origin feature/название-фичи
```

### 8. Создайте Pull Request
Опишите изменения, которые вы внесли, и дождитесь ревью.

## ⚙️ Настройка окружения

### Локальная разработка

1. **Требования:**
   - Python 3.10+
   - PostgreSQL (опционально, можно использовать SQLite)
   - Docker (опционально)

2. **Установка:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/Mac
   ```

3. **Запуск:**
   ```bash
   python init_db.py  # Создание админа
   uvicorn app.main:app --reload
   ```

### Разработка с Docker

```bash
docker-compose up --build
```

## 📝 Стандарты кода

### Python

- **PEP 8** — следуйте официальному стилю Python
- **Типизация** — используйте аннотации типов
- **Docstrings** — документируйте все публичные функции и классы
- **Импорты** — группируйте импорты: стандартная библиотека → сторонние → локальные

**Пример:**
```python
from datetime import datetime
from typing import List, Optional

from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session

from app.database import get_db
from app.models import Room


@router.get("/rooms/", response_model=List[RoomOut])
def get_rooms(skip: int = 0, limit: int = 100, db: Session = Depends(get_db)):
    """
    Получение списка всех пространств.
    
    Args:
        skip: Количество пропускаемых записей
        limit: Максимальное количество записей
        db: Сессия базы данных
    
    Returns:
        Список пространств
    """
    return db.query(Room).offset(skip).limit(limit).all()
```

### JavaScript

- **ES6+** — используйте современные возможности JavaScript
- **Константы** — используйте `const` по умолчанию, `let` только при необходимости
- **Асинхронность** — используйте `async/await` вместо `.then()`
- **Обработка ошибок** — всегда используйте `try/catch` для асинхронных операций

**Пример:**
```javascript
async function loadRooms() {
    try {
        const response = await fetch(`${API_URL}/rooms/`);
        if (!response.ok) {
            throw new Error('Ошибка загрузки комнат');
        }
        const rooms = await response.json();
        renderRooms(rooms);
    } catch (error) {
        console.error('Ошибка:', error);
        showError('Не удалось загрузить комнаты');
    }
}
```

### CSS

- **CSS-переменные** — используйте переменные для цветов и размеров
- **БЭМ-нотация** — используйте понятные имена классов
- **Адаптивность** — всегда тестируйте на мобильных устройствах

## 🧪 Тестирование

### Запуск тестов

```bash
pytest tests/test_api.py -v
```

### Добавление новых тестов

1. Добавьте тест в файл `tests/test_api.py`
2. Используйте фикстуры для создания тестовых данных
3. Тест должен быть независимым от других тестов
4. Тест должен проверять одну конкретную функциональность

**Пример:**
```python
def test_create_room_with_equipment(db_session, admin_token):
    """Тест создания комнаты с оборудованием"""
    headers = {"Authorization": f"Bearer {admin_token}"}
    
    room_data = {
        "name": "Test Room",
        "capacity": 10,
        "equipment": ["projector", "whiteboard"]
    }
    
    response = client.post("/rooms/", json=room_data, headers=headers)
    
    assert response.status_code == 201
    data = response.json()
    assert data["equipment"] == ["projector", "whiteboard"]
```

### Покрытие кода

Стремитесь к покрытию ключевой бизнес-логики:

- Проверка пересечений бронирований
- Аутентификация и авторизация

##  Процесс ревью

### Что проверяется:

1. **Функциональность** — код работает корректно
2. **Тесты** — добавлены тесты для новых функций
3. **Стиль** — код следует стандартам проекта
4. **Документация** — обновлены README и docstrings
5. **Безопасность** — нет уязвимостей
6. **Производительность** — нет очевидных проблем с производительностью

### Время ревью

- Критические исправления: до 24 часов
- Новые функции: до 48 часов
- Мелкие изменения: до 72 часов

## 🐛 Сообщение об ошибках

### Как сообщить об ошибке

1. Проверьте, что ошибка ещё не сообщалась в Issues
2. Создайте новый Issue с меткой `bug`
3. Опишите проблему подробно:
   - Что вы делали
   - Что ожидали увидеть
   - Что произошло на самом деле
   - Логи ошибок (если есть)

### Шаблон Issue

```markdown
## Описание
Краткое описание проблемы

## Шаги воспроизведения
1. ...
2. ...
3. ...

## Ожидаемое поведение
Что должно было произойти

## Фактическое поведение
Что произошло на самом деле

## Логи
```
[вставьте логи здесь]
```

## Окружение
- OS: Windows/Mac/Linux
- Python: 3.x
- Браузер: Chrome/Firefox/Safari
```
