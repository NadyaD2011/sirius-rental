#  История изменений

Все значимые изменения в проекте «Сириус.Аренда» будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версионирование соответствует [Semantic Versioning](https://semver.org/lang/ru/).


## [1.0.0]

###  Добавлено
- Полный веб-интерфейс с адаптивным дизайном и тёмной темой
- Визуальный календарь бронирований (FullCalendar.js)
- Дашборд со статистикой для администраторов
- Умный поиск свободных комнат с фильтрами (время, вместимость, оборудование)
- Система ролей (user/admin) с разграничением прав
- Восстановление пароля по имени пользователя
- Смена пароля текущего пользователя
- Редактирование профиля пользователя
- Управление пользователями для админов (создание, редактирование, удаление, сброс паролей)
- Принудительная отмена бронирований администратором
- Фильтрация бронирований по статусу, дате, оборудованию и вместимости
- Отображение размера комнаты в списке бронирований
- Анимация переключения темы
- Скрипт инициализации базы данных с созданием админа
- Docker-контейнеризация с PostgreSQL
- Healthcheck для PostgreSQL в Docker Compose


###  Исправлено
- Ошибка совместимости bcrypt 4.0+ с passlib (зафиксирована версия 3.2.2)
- Проблема с порядком запуска контейнеров Docker (добавлен healthcheck)

## [0.9.0]

###  Добавлено
- Адаптивный дизайн для мобильных устройств
- Тёмная тема с переключателем
- Расширенная фильтрация на странице "Все бронирования"
- Редактирование комнат (полное CRUD)
- История бронирований с возможностью просмотра отменённых
- Модальное окно профиля пользователя


## [0.8.0]

### Добавлено
- Интеграция FullCalendar.js для визуализации расписания
- Поддержка просмотра бронирований за период
- Эндпоинт `/rooms/available` с фильтрацией по оборудованию
- Опциональный параметр `capacity` для поиска свободных комнат
- Кастомные стили для календаря

###  Изменено
- Обновлён эндпоинт `/bookings/room/{id}` для поддержки диапазонов дат
- Улучшена валидация параметров поиска

---

## [0.7.0]

###  Добавлено
- JWT-аутентификация с OAuth2 flow
- Разделение ролей (user/admin)
- Защита эндпоинтов по ролям
- Создание администраторов через API
- Список всех пользователей для админов
- Изменение ролей пользователей
- Удаление пользователей
- Сброс паролей администратором
- Просмотр всех бронирований системы
- Принудительная отмена бронирований

###  Изменено
- Эндпоинты создания/редактирования/удаления комнат требуют роль admin
- Бронирования привязаны к конкретному пользователю
- Отмена бронирования доступна только владельцу или админу

---

## [0.6.0]

###  Добавлено
- Юнит-тесты (5 тестов) с pytest
- Фикстуры для тестирования
- Тестирование проверки пересечений бронирований
- Тестирование поиска свободных комнат
- Файл `pytest.ini` для конфигурации тестов
- Файл `.env` для переменных окружения
- Поддержка PostgreSQL через переменные окружения

### Изменено
- Обновлены схемы Pydantic для совместимости с v2
- Исправлены предупреждения SQLAlchemy и Pydantic
- Улучшена структура проекта

## [0.5.0]

###  Добавлено
- Базовый CRUD для пространств (rooms)
- CRUD для бронирований (bookings)
- Проверка пересечений бронирований
- Обработка ошибок с правильными HTTP-статусами
- Swagger-документация (автоматическая)
- CORS для фронтенда
- Простой веб-интерфейс (HTML/CSS/JS)
- Форма бронирования
- Список комнат с карточками

### Изменено
- Создана реляционная структура базы данных
- Добавлена валидация входных данных

## [0.1.0]

###  Добавлено
- Начальная структура проекта
- Настройка FastAPI
- Подключение SQLAlchemy
- Базовые модели данных
- Pydantic схемы для валидации
