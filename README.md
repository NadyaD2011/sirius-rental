##  О проекте

**Сириус.Аренда** — это полноценный сервис для управления переговорными комнатами и учебными пространствами Университета «Сириус». Проект позволяет студентам, преподавателям и гостям бронировать помещения, проверять их доступность и просматривать расписание.

Проект разработан в рамках конкурса **«Всероссийский ИТ-раунд»** по треку **«Разработка и управление программным обеспечением»**.


##  Возможности

###  Для пользователей
-  Регистрация и вход в систему (JWT-аутентификация)
-  Поиск и просмотр доступных пространств
-  Бронирование комнаты на выбранное время
-  Просмотр своих бронирований (активных и отменённых)
-  Отмена собственных бронирований
-  Умный поиск свободных комнат по фильтрам (время, вместимость, оборудование)
-  Смена пароля и восстановление доступа

### ️ Для администраторов
- Всё, что доступно пользователям, плюс:
- Создание, редактирование и удаление пространств
- Управление пользователями (создание аккаунтов, смена ролей, сброс паролей)
- Просмотр всех бронирований системы с расширенной фильтрацией
- Принудительная отмена любых бронирований
- Дашборд со статистикой (загрузка комнат, популярные пространства)


##  Стек технологий и обоснование выбора

| Технология | Назначение | Почему выбрано |
|------------|------------|----------------|
| **Python 3.11** | Язык программирования | Высокая скорость разработки, читаемость, богатая экосистема |
| **FastAPI** | Web-фреймворк | Автоматическая Swagger-документация, асинхронность, валидация через Pydantic, высокая производительность |
| **SQLAlchemy 2.0** | ORM для работы с БД | Абстракция над БД, поддержка PostgreSQL и SQLite, удобная миграция между ними |
| **PostgreSQL 16** | Основная БД | Надёжность, транзакции, промышленный стандарт |
| **SQLite** | БД для разработки | Встроена в Python, не требует установки |
| **JWT (python-jose)** | Аутентификация | Stateless-токены, удобны для REST API, не требуют хранения сессий |
| **Passlib + bcrypt** | Хеширование паролей | Безопасное хранение паролей, устойчивость к brute-force |
| **Docker + Compose** | Контейнеризация | Изоляция зависимостей, упрощение развёртывания |
| **pytest** | Тестирование | Простой синтаксис, фикстуры, покрытие бизнес-логики |

## Установка и запуск

### Вариант 1: Локальный запуск (для разработки)

#### Требования
- Python 3.10+
- pip

#### Шаги

**1. Клонирование репозитория:**
```bash
git clone https://github.com/NadyaD2011/sirius-rental.git
cd sirius-rental
```

**2. Создание виртуального окружения:**
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/macOS
python3 -m venv venv
source venv/bin/activate
```

**3. Установка зависимостей:**
```bash
pip install -r requirements.txt
```

**4. Инициализация базы данных:**
```bash
python init_db.py
```
> Скрипт создаст таблицы и первого администратора: **`admin` / `admin123`**

**5. Запуск сервера:**
```bash
uvicorn app.main:app --reload
```

**6. Открытие приложения:**
-  **Swagger-документация API:** http://127.0.0.1:8000/docs
-  **Альтернативная документация (ReDoc):** http://127.0.0.1:8000/redoc
-  **Веб-интерфейс(запускать паралельно во второй консоли):**
  ```bash
  cd frontend
  python -m http.server 5500
  ```
  Затем откройте http://localhost:5500


### Вариант 2: Запуск через Docker

#### Требования
- Docker Desktop
- Docker Compose

#### Шаги

**1. Сборка и запуск контейнеров:**
```bash
docker-compose up --build
```

**2. Создание администратора (в новом терминале):**
```bash
docker-compose exec backend python init_db.py
```

**3. Открытие приложения:**
-  **Swagger API:** http://localhost:8000/docs
-  **Веб-интерфейс:** http://localhost:5500 (запустить отдельно)

**4. Остановка:**
```bash
docker-compose down
```

##  Документация API

После запуска сервера полная интерактивная документация доступна по адресу **http://127.0.0.1:8000/docs** (Swagger UI).

###  Аутентификация

Большинство эндпоинтов требуют JWT-токен, который передаётся в заголовке:
```
Authorization: Bearer <ваш_токен>
```

#### `POST /users/login` — Вход в систему
**Формат:** `application/x-www-form-urlencoded` (OAuth2)

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `username` | string | Обязательный | Имя пользователя |
| `password` | string | Обязательный | Пароль |

**Ответ 200:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer"
}
```

**Ответ 401:**
```json
{ "detail": "Неверное имя пользователя или пароль" }
```


###  Пользователи

#### `POST /users/register` — Регистрация
**Тело запроса (JSON):**
```json
{
  "username": "student",
  "password": "secret123"
}
```

**Ответ 201:**
```json
{
  "id": 2,
  "username": "student",
  "role": "user"
}
```

**Ответ 409:** `{"detail": "Пользователь уже существует"}`

#### `GET /users/me` — Текущий пользователь
*Требует авторизации.*

**Ответ 200:**
```json
{
  "id": 1,
  "username": "admin",
  "role": "admin"
}
```

#### `POST /users/change-password` — Смена пароля
*Требует авторизации.*

**Тело запроса:**
```json
{
  "old_password": "secret123",
  "new_password": "newpassword456"
}
```

#### `POST /users/recover-password` — Восстановление пароля
**Тело запроса:**
```json
{
  "username": "student",
  "new_password": "newpassword456"
}
```

#### `POST /users/create-account` — Создание аккаунта с ролью
*Только для администраторов.*

**Тело запроса:**
```json
{
  "username": "teacher",
  "password": "teacher123",
  "role": "admin"
}
```

#### `GET /users/list` — Список всех пользователей
*Только для администраторов.*

#### `PUT /users/{user_id}/role` — Изменение роли
*Только для администраторов.*

#### `PUT /users/{user_id}/username` — Изменение имени
*Пользователь может менять своё имя, админ — любое.*

#### `PUT /users/{user_id}/reset-password` — Сброс пароля
*Только для администраторов.*

#### `DELETE /users/{user_id}` — Удаление пользователя
*Только для администраторов. Нельзя удалить себя.*

###  Пространства

#### `POST /rooms/` — Создание пространства
*Только для администраторов.*

**Тело запроса:**
```json
{
  "name": "Переговорная №1",
  "capacity": 10,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "name": "Переговорная №1",
  "capacity": 10,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

#### `GET /rooms/` — Список всех пространств
*Публичный эндпоинт.*

**Параметры (query, опционально):**
- `skip` (int) — пропуск записей
- `limit` (int) — максимум записей

**Ответ 200:**
```json
[
  {
    "id": 1,
    "name": "Переговорная №1",
    "capacity": 10,
    "equipment": ["проектор", "доска"]
  },
  {
    "id": 2,
    "name": "Аудитория 305",
    "capacity": 30,
    "equipment": ["проектор"]
  }
]
```

#### `GET /rooms/{room_id}` — Детали пространства

#### `PUT /rooms/{room_id}` — Редактирование
*Только для администраторов.*

#### `DELETE /rooms/{room_id}` — Удаление
*Только для администраторов.*

**Ответ 204:** No Content

#### `GET /rooms/available` — Поиск свободных пространств

*Публичный эндпоинт.*

**Параметры (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `start` | datetime | Обязательный | Время начала (ISO 8601) |
| `end` | datetime | Обязательный | Время окончания (ISO 8601) |
| `capacity` | int | Не обязательный | Минимальная вместимость |
| `equipment` | List[str] | Не обязательный | Требуемое оборудование |

**Пример запроса:**
```
GET /rooms/available?start=2026-07-15T10:00:00&end=2026-07-15T11:00:00&capacity=5&equipment=проектор
```

**Ответ 200:**
```json
[
  {
    "id": 2,
    "name": "Аудитория 305",
    "capacity": 30,
    "equipment": ["проектор"]
  }
]
```

**Ответ 400:** `{"detail": "Время окончания должно быть позже времени начала"}`

### Бронирования

#### `POST /bookings/` — Создание бронирования
*Требует авторизации.*

**Тело запроса:**
```json
{
  "room_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00"
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "room_id": 1,
  "user_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00",
  "status": "active"
}
```

**Ответ 400:** `{"detail": "Время окончания должно быть позже времени начала"}`

**Ответ 404:** `{"detail": "Пространство не найдено"}`

**Ответ 409 (конфликт):** `{"detail": "Пространство уже занято в указанное время"}`

> **Алгоритм проверки пересечений:** два бронирования пересекаются, если `start1 < end2 AND start2 < end1`. Бронирования «стык в стык» (например, 10:00–11:00 и 11:00–12:00) **не считаются пересечением**.

#### `DELETE /bookings/{booking_id}` — Отмена бронирования
*Требует авторизации. Владелец или админ.*

**Ответ 204:** No Content

**Ответ 403:** `{"detail": "Нет прав для отмены"}`

#### `GET /bookings/my` — Мои бронирования
*Требует авторизации.*

**Параметры (query):**
- `include_cancelled` (bool, default: `false`) — включить отменённые

#### `GET /bookings/all` — Все бронирования системы
*Только для администраторов.*

#### `GET /bookings/room/{room_id}` — Расписание комнаты

**Параметры (query):**
- `date` (YYYY-MM-DD) — для одного дня
- или `start_date` + `end_date` — для диапазона


##  База данных

### Выбор системы хранения

Проект поддерживает **две СУБД** через абстракцию SQLAlchemy:
- **SQLite** — для локальной разработки (файл `rental.db`)
- **PostgreSQL 16** — для продакшн-развертывания (Docker)

Переключение между ними происходит через переменную окружения `DATABASE_URL` без изменения кода.

##  Контейнеризация

Проект полностью упакован в Docker-контейнеры с использованием Docker Compose.

**Запуск одной командой:**
```bash
docker-compose up --build
```

**Преимущества:**
- Не требует установки Python и PostgreSQL
- Работает на любой ОС (Windows/macOS/Linux)
- Изоляция зависимостей
- Готовность к продакшн-развертыванию
- Healthcheck для PostgreSQL гарантирует корректный порядок запуска


##  Безопасность

- **JWT-токены** с временем жизни 60 минут
- **Хеширование паролей** алгоритмом bcrypt
- **Ролевая модель** (user/admin) с разграничением прав на уровне эндпоинтов
- **Валидация входных данных** через Pydantic-схемы
- **CORS** настроен для безопасного взаимодействия с фронтендом

##  Обработка ошибок

API возвращает понятные сообщения об ошибках с корректными HTTP-статусами:

| Код | Значение | Пример использования |
|-----|----------|----------------------|
| 200 | OK | Успешный GET/PUT |
| 201 | Created | Успешное создание ресурса |
| 204 | No Content | Успешное удаление |
| 400 | Bad Request | Некорректные данные |
| 401 | Unauthorized | Неверный токен или учётные данные |
| 403 | Forbidden | Недостаточно прав (например, user пытается удалить комнату) |
| 404 | Not Found | Ресурс не найден |
| 409 | Conflict | Пространство уже занято в выбранное время |
