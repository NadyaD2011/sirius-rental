## 📖 О проекте

**Сириус.Аренда** — это полноценный сервис для управления переговорными комнатами и учебными пространствами Университета «Сириус». Проект позволяет студентам, преподавателям и гостям бронировать помещения, проверять их доступность и просматривать расписание.

Проект разработан в рамках конкурса **«Всероссийский ИТ-раунд»** по треку **«Разработка и управление программным обеспечением»**.

## ✨ Возможности

### 👤 Для пользователей
-  Регистрация и вход в систему (JWT-аутентификация)
- 🔍 Поиск и просмотр доступных пространств
- 📅 Бронирование комнаты на выбранное время
-  Просмотр своих бронирований (активных и отменённых)

##  Стек технологий и обоснование выбора

| Технология | Назначение | Почему выбрано |
|------------|------------|----------------|
| **Python 3.11** | Язык программирования | Высокая скорость разработки, читаемость, богатая экосистема |

## 🚀 Установка и запуск

### Вариант 1: Локальный запуск (для разработки)

#### Требования
- Python 3.10+
- pip

#### Шаги

**1. Клонирование репозитория:**
```bash
git clone https://github.com/ВАШ_ЮЗЕРНЕЙМ/sirius-rental.git
cd sirius-rental
```

**2. Создание виртуального окружения:**
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/macOS
python3 -m venv venv
source venv/bin/activate
```

**3. Установка зависимостей:**
```bash
pip install -r requirements.txt
```

**4. Инициализация базы данных:**
```bash
python init_db.py
```
> Скрипт создаст таблицы и первого администратора: **`admin` / `admin123`**

**5. Запуск сервера:**
```bash
uvicorn app.main:app --reload
```

**6. Открытие приложения:**
- 📘 **Swagger-документация API:** http://127.0.0.1:8000/docs
- 📗 **Альтернативная документация (ReDoc):** http://127.0.0.1:8000/redoc
-  **Веб-интерфейс:**
  ```bash
  cd frontend
  python -m http.server 5500
  ```
  Затем откройте http://localhost:5500

## 📚 Документация API

После запуска сервера полная интерактивная документация доступна по адресу **http://127.0.0.1:8000/docs** (Swagger UI).

### 🔐 Аутентификация

Большинство эндпоинтов требуют JWT-токен, который передаётся в заголовке:
```
Authorization: Bearer <ваш_токен>
```

#### `POST /users/login` — Вход в систему
**Формат:** `application/x-www-form-urlencoded` (OAuth2)

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `username` | string | ✅ | Имя пользователя |
| `password` | string | ✅ | Пароль |

**Ответ 200:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer"
}
```

**Ответ 401:**
```json
{ "detail": "Неверное имя пользователя или пароль" }
```

**Тело запроса:**

*Только для администраторов.*

#### `DELETE /users/{user_id}` — Удаление пользователя

### 🏢 Пространства

#### `POST /rooms/` — Создание пространства
*Только для администраторов.*

**Тело запроса:**
```json
{
  "name": "Переговорная №1",
  "capacity": 10,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "name": "Переговорная №1",
  "capacity": 10,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

#### `GET /rooms/` — Список всех пространств
*Публичный эндпоинт.*

**Параметры (query, опционально):**
- `skip` (int) — пропуск записей
- `limit` (int) — максимум записей

**Ответ 200:**
```json
[
  {
    "id": 1,
    "name": "Переговорная №1",
    "capacity": 10,
    "equipment": ["проектор", "доска"]
  },
  {
    "id": 2,
    "name": "Аудитория 305",
    "capacity": 30,
    "equipment": ["проектор"]
  }
]
```

#### `GET /rooms/{room_id}` — Детали пространства

#### `PUT /rooms/{room_id}` — Редактирование
*Только для администраторов.*

#### `DELETE /rooms/{room_id}` — Удаление
*Только для администраторов.*

**Ответ 204:** No Content

#### `GET /rooms/available` — Поиск свободных пространств ⭐
*Публичный эндпоинт. Ключевая бизнес-логика.*

**Параметры (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `start` | datetime | ✅ | Время начала (ISO 8601) |
| `end` | datetime | ✅ | Время окончания (ISO 8601) |
| `capacity` | int |  | Минимальная вместимость |
| `equipment` | List[str] | ❌ | Требуемое оборудование |

**Пример запроса:**
```
GET /rooms/available?start=2026-07-15T10:00:00&end=2026-07-15T11:00:00&capacity=5&equipment=проектор
```

**Ответ 200:**
```json
[
  {
    "id": 2,
    "name": "Аудитория 305",
    "capacity": 30,
    "equipment": ["проектор"]
  }
]
```

**Ответ 400:** `{"detail": "Время окончания должно быть позже времени начала"}`

### 📅 Бронирования

#### `POST /bookings/` — Создание бронирования
*Требует авторизации.*

**Тело запроса:**
```json
{
  "room_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00"
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "room_id": 1,
  "user_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00",
  "status": "active"
}
```

**Ответ 400:** `{"detail": "Время окончания должно быть позже времени начала"}`

**Ответ 404:** `{"detail": "Пространство не найдено"}`

**Ответ 409 (конфликт):** `{"detail": "Пространство уже занято в указанное время"}`

> **Алгоритм проверки пересечений:** два бронирования пересекаются, если `start1 < end2 AND start2 < end1`. Бронирования «стык в стык» (например, 10:00–11:00 и 11:00–12:00) **не считаются пересечением**.

#### `DELETE /bookings/{booking_id}` — Отмена бронирования
*Требует авторизации. Владелец или админ.*

**Ответ 204:** No Content

**Ответ 403:** `{"detail": "Нет прав для отмены"}`

#### `GET /bookings/my` — Мои бронирования
*Требует авторизации.*

**Параметры (query):**
- `include_cancelled` (bool, default: `false`) — включить отменённые

#### `GET /bookings/all` — Все бронирования системы
*Только для администраторов.*

#### `GET /bookings/room/{room_id}` — Расписание комнаты

**Параметры (query):**
- `date` (YYYY-MM-DD) — для одного дня
- или `start_date` + `end_date` — для диапазона

## 🗄️ База данных

### Схема (реляционная)

```
─────────────────┐       ┌──────────────────┐       ┌─────────────────┐
│     users       │       │    bookings      │       │     rooms       │
├─────────────────┤       ├──────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────│ user_id (FK)     │       │ id (PK)         │
│ username (UQ)   │       │ room_id (FK)     │──────►│ name            │
│ hashed_password │       │ start_time       │       │ capacity        │
│ role            │       │ end_time         │       │ equipment       │
└─────────────────┘       │ status           │       └─────────────────┘
                          └──────────────────┘
```

### Выбор системы хранения

Проект поддерживает **две СУБД** через абстракцию SQLAlchemy:
- **SQLite** — для локальной разработки (файл `rental.db`)
- **PostgreSQL 16** — для продакшн-развертывания (Docker)

Переключение между ними происходит через переменную окружения `DATABASE_URL` без изменения кода.

## 📊 Обработка ошибок

API возвращает понятные сообщения об ошибках с корректными HTTP-статусами:

| Код | Значение | Пример использования |
|-----|----------|----------------------|
| 200 | OK | Успешный GET/PUT |
| 201 | Created | Успешное создание ресурса |
| 204 | No Content | Успешное удаление |
| 400 | Bad Request | Некорректные данные (`end_time < start_time`) |
| 401 | Unauthorized | Неверный токен или учётные данные |
| 403 | Forbidden | Недостаточно прав (например, user пытается удалить комнату) |
| 404 | Not Found | Ресурс не найден |
| 409 | Conflict | Пространство уже занято в выбранное время |
