## 📝 Описание проекта

**Сириус.Аренда** — это сервис, который позволяет студентам, преподавателям и гостям Университета бронировать переговорные комнаты и учебные пространства. Проект реализует полноценный REST API с системой аутентификации, ролевой моделью и веб-интерфейсом.

### Основные возможности:
- ✅ Управление пространствами (CRUD)
- ✅ Бронирование с проверкой доступности
- ✅ Поиск свободных комнат по фильтрам
- ✅ JWT-аутентификация с ролями (user/admin)
- ✅ Визуализация расписания (календарь)
- ✅ Админ-панель для управления пользователями
- ✅ Адаптивный веб-интерфейс с тёмной темой

##  Стек технологий

| Технология | Назначение | Обоснование выбора |
|------------|------------|-------------------|
| **Python 3.11** | Язык программирования | Высокая скорость разработки, читаемость кода |

| **SQLite** | База для разработки | Встроена в Python, не требует установки |
| **JWT (python-jose)** | Аутентификация | Stateless-токены, удобны для REST API |

| **Docker + Docker Compose** | Контейнеризация | Упрощает развёртывание, изоляция зависимостей |

## 🚀 Установка и запуск

### Локальный запуск

#### 1. Требования
- Python 3.10+
- pip

#### 2. Клонирование репозитория
```bash
git clone <url-репозитория>
cd sirius-rental
```

#### 3. Создание виртуального окружения
```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate
```

#### 4. Установка зависимостей
```bash
pip install -r requirements.txt
```

#### 5. Инициализация базы данных
```bash
python init_db.py
```
*Скрипт создаст первого администратора: `admin` / `admin123`*

#### 6. Запуск сервера
```bash
uvicorn app.main:app --reload
```

#### 7. Открытие приложения
- **API документация (Swagger):** http://127.0.0.1:8000/docs
- **Альтернативная документация (ReDoc):** http://127.0.0.1:8000/redoc
- **Веб-интерфейс:** откройте `frontend/index.html` в браузере или запустите:
  ```bash
  cd frontend
  python -m http.server 5500
  ```
  Затем откройте http://localhost:5500

---

- **API:** http://localhost:8000/docs

## 📚 Документация API

Все эндпоинты доступны через Swagger UI по адресу `/docs` после запуска сервера.

### Аутентификация

Большинство эндпоинтов требуют JWT-токен. Токен передаётся в заголовке:
```
Authorization: Bearer <token>
```

#### `POST /users/login`
Вход в систему (OAuth2 form-data).

**Параметры (form-data):**
- `username` (string) — имя пользователя
- `password` (string) — пароль

**Ответ 200:**
```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "bearer"
}
```

---

### Пользователи

#### `POST /users/register`
Регистрация нового пользователя.

**Тело запроса (JSON):**
```json
{
  "username": "string (3-50 символов)",
  "password": "string (мин. 6 символов)"
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "username": "nadya",
  "role": "user"
}
```

#### `GET /users/me`
Информация о текущем пользователе. *Требует авторизации.*

**Ответ 200:**
```json
{
  "id": 1,
  "username": "nadya",
  "role": "user"
}
```

#### `POST /users/change-password`
Смена пароля текущего пользователя. *Требует авторизации.*

**Тело запроса (JSON):**
```json
{
  "old_password": "string",
  "new_password": "string (мин. 6 символов)"
}
```

#### `POST /users/recover-password`
Восстановление пароля по имени пользователя.

**Тело запроса (JSON):**
```json
{
  "username": "string",
  "new_password": "string (мин. 6 символов)"
}
```

#### `POST /users/create-account`
Создание аккаунта с выбором роли. *Только для администраторов.*

**Тело запроса (JSON):**
```json
{
  "username": "string",
  "password": "string (мин. 6 символов)",
  "role": "user или admin"
}
```

#### `GET /users/list`
Список всех пользователей. *Только для администраторов.*

#### `PUT /users/{user_id}/role`
Изменение роли пользователя. *Только для администраторов.*

#### `PUT /users/{user_id}/username`
Изменение имени пользователя.

#### `PUT /users/{user_id}/reset-password`
Сброс пароля пользователя. *Только для администраторов.*

#### `DELETE /users/{user_id}`
Удаление пользователя. *Только для администраторов.*

### Пространства

#### `POST /rooms/`
Создание нового пространства. *Только для администраторов.*

**Тело запроса (JSON):**
```json
{
  "name": "Переговорная №1",
  "capacity": 10,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "name": "Переговорная №1",
  "capacity": 10,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

#### `GET /rooms/`
Получение списка всех пространств. *Публичный эндпоинт.*

**Ответ 200:**
```json
[
  {
    "id": 1,
    "name": "Переговорная №1",
    "capacity": 10,
    "equipment": ["проектор", "доска"]
  }
]
```

#### `GET /rooms/{room_id}`
Получение детальной информации о пространстве.

#### `PUT /rooms/{room_id}`
Редактирование пространства. *Только для администраторов.*

#### `DELETE /rooms/{room_id}`
Удаление пространства. *Только для администраторов.*

#### `GET /rooms/available`
Поиск свободных пространств. *Публичный эндпоинт.*

**Параметры запроса (query):**
- `start` (datetime, обязательный) — время начала в ISO-формате
- `end` (datetime, обязательный) — время окончания в ISO-формате
- `capacity` (int, опциональный) — минимальная вместимость
- `equipment` (List[str], опциональный) — список требуемого оборудования

**Пример запроса:**
```
GET /rooms/available?start=2026-07-15T10:00:00&end=2026-07-15T11:00:00&capacity=5&equipment=проектор
```

**Ответ 200:**
```json
[
  {
    "id": 2,
    "name": "Аудитория 305",
    "capacity": 20,
    "equipment": ["проектор", "доска"]
  }
]
```

---

### Бронирования

#### `POST /bookings/`
Создание бронирования. *Требует авторизации.*

**Тело запроса (JSON):**
```json
{
  "room_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00"
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "room_id": 1,
  "user_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00",
  "status": "active"
}
```

**Возможные ошибки:**
- `400 Bad Request` — некорректные данные (например, end_time < start_time)
- `404 Not Found` — пространство не найдено
- `409 Conflict` — пространство уже занято в указанное время

#### `DELETE /bookings/{booking_id}`
Отмена бронирования. *Требует авторизации (владелец или админ).*

**Ответ 204 No Content**

#### `GET /bookings/my`
Мои бронирования. *Требует авторизации.*

**Параметры запроса (query):**
- `include_cancelled` (bool, по умолчанию false) — включить отменённые

#### `GET /bookings/all`
Все бронирования системы. *Только для администраторов.*

#### `GET /bookings/room/{room_id}`
Бронирования конкретной комнаты.

**Параметры запроса (query):**
- `date` (YYYY-MM-DD) — для одного дня
- или `start_date` + `end_date` — для диапазона

## 🧪 Примеры использования

### Сценарий 1: Регистрация и бронирование

```bash
# 1. Регистрация
curl -X POST "http://127.0.0.1:8000/users/register" \
  -H "Content-Type: application/json" \
  -d '{"username": "student", "password": "secret123"}'

# 2. Вход
curl -X POST "http://127.0.0.1:8000/users/login" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=student&password=secret123"

# 3. Получение списка комнат
curl "http://127.0.0.1:8000/rooms/"

# 4. Бронирование (с токеном из шага 2)
curl -X POST "http://127.0.0.1:8000/bookings/" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"room_id": 1, "start_time": "2026-07-15T10:00:00", "end_time": "2026-07-15T11:00:00"}'
```

### Сценарий 2: Поиск свободных комнат

```bash
curl "http://127.0.0.1:8000/rooms/available?start=2026-07-15T10:00:00&end=2026-07-15T11:00:00&capacity=5"
```
